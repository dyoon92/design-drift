# Welcome to Drift

This Storybook is the living component library for **StorageOS** — and the integration point for [Drift](https://catchdrift.ai), the design system coverage layer built for teams shipping with AI.

Every component here is tracked by Drift. When AI builds a new page, Drift checks what it produced against this library, gives it a coverage score, and flags anything that drifted.

---

## How Drift + Storybook fit together

Storybook is where your components live. Drift is what keeps AI honest about using them.

When a developer or AI tool builds a screen, Drift scans the rendered component tree in real time — it doesn't read source code, it reads what's actually on the page. Any component that isn't in this Storybook registry gets flagged as a gap.

The result: a coverage score per page, a list of what drifted, and a one-click path to fix it or promote it into the design system.

## The design token layer

Design tokens (colors, spacing, typography, radii) live in `src/tokens/variables.css` and are synced directly from Figma via `npm run figma-sync`. Every component in this library uses `var(--ds-color-*)` and `var(--ds-spacing-*)` — never hardcoded values.

When Drift finds a component using `#7d52f7` instead of `var(--ds-color-primary)`, it flags it as a token violation. Same component, same pixel value — but the design system doesn't own it yet.

## Running Drift on this app

```bash
npm run dev        # start the app on :5173
                   # press D to open the Drift overlay
npm run storybook  # this catalog on :6006
npm run figma-sync # pull latest tokens from Figma
npm run drift-check # CI coverage check
```

Press **D** in the running app to see the coverage overlay — green borders for DS components, red for gaps.
