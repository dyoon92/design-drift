# The End-to-End Drift Flow

This page explains how Drift fits into the full design → build → ship cycle — and why each step matters.

---

## The problem Drift solves

AI coding tools are fast. Really fast. In the time it takes to write a ticket, Cursor or Claude Code has already built the screen. But speed without guardrails means components get invented from scratch, tokens get hardcoded, and the design system stops being the source of truth.

Drift is the layer that catches it — in the browser, in the IDE, and in CI — before it ships.

---

## Step 1 — Figma stays the design source of truth

Designers define components, colors, spacing, and typography in Figma. Tokens Studio syncs those values to the codebase automatically via Style Dictionary.

```bash
npm run figma-sync   # Figma → src/tokens/variables.css
```

Every color, spacing value, and radius in this component library comes from that sync. Nothing is hardcoded.

---

## Step 2 — Components live in Storybook

Every reusable component is documented here. When AI tools build new screens, this catalog is what they're supposed to reference.

`CLAUDE.md` in the repo root makes that explicit — it lists every available component, where it lives, and which tokens to use. Claude Code, Cursor, and Windsurf all read it before generating code.

If a component doesn't exist in Storybook yet, the AI is instructed to output a `⚠️ Missing component` warning instead of inventing one from scratch. That warning becomes a design request.

---

## Step 3 — AI ships fast (and introduces drift)

A developer or PM describes a screen. The AI builds it in seconds. Most of it is correct — but somewhere in there, a custom card got invented, a hex color got hardcoded, or a spacing value was eyeballed.

This is normal. The goal isn't to stop AI from moving fast. It's to know exactly what it got wrong.

---

## Step 4 — Press D, see what drifted

In the running app (`npm run dev`), press **D** to open the Drift overlay. Every component on the page gets a border:

- **Green** — registered in your design system
- **Red** — custom or unknown (a gap)
- **Orange** — present in the DS but a token is hardcoded somewhere

The coverage score for that page appears in the top-right panel. A score of 100% means every component is from the system. A score of 60% means 40% of the page was built outside it.

---

## Step 5 — Fix gaps from the browser or IDE

From the overlay, you can:

- **Promote** a custom component — generates a Cursor prompt to rebuild it from Figma, then Drift tracks it
- **Create a Figma brief** — generates a structured spec (props, states, token requirements) so a designer knows exactly what to build next
- **File a Jira ticket** — pre-filled with component name, screenshot, and occurrence count
- **Queue a fix for Claude Code** — the MCP server surfaces `drift_gaps` and `drift_suggest` directly in Claude Code or Cursor, so you can apply fixes without leaving the editor

---

## Step 6 — CI enforces the threshold

The GitHub Action runs `npm run drift-check` on every PR. It posts a coverage delta as a PR comment — what percentage was in the DS before, what it is now, and what changed.

If coverage drops below your configured threshold, the check fails. Drift becomes part of the definition of done.

## The full picture

```
Figma (source of truth)
    │
    │  npm run figma-sync
    ▼
CSS tokens → Storybook components → CLAUDE.md
                                        │
                                        │  AI reads rules, builds screens
                                        ▼
                               Running app (npm run dev)
                                        │
                                        │  press D
                                        ▼
                               Drift overlay — coverage score, gap list
                                        │
                          ┌─────────────┴─────────────┐
                          ▼                           ▼
                    Fix in IDE                 Promote to DS
                  (MCP tools)             (Figma brief + Jira ticket)
                          │                           │
                          └─────────────┬─────────────┘
                                        ▼
                                   PR + CI check
                              (drift-check GitHub Action)
```
